JavaScript
Selenium-Chai-Mocha
Test Framework

This is a Proof-of-Concept test framework written in JavaScript that uses Selenium Webdriver, Chai, and Mocha.
It's designed to test the Library demo app from the EmberJS tutorial.

This goes over the basic features of Selenium Webdriver, Chai, and Mocha.

  • Selenium Webdriver - a simple and concise programming interface that drives the browser via object-oriented API.
  • Chai - a BDD/TDD assertion library for node and paired usually with the JavaScript testing framework.
  • Mocha - a JavaScript test framework that runs on NodeJS. It's best used for async testing and runs serially.

This tutorial was done on Mac OS. I don't have an explicit tutorial for Windows. But, I'll post a YouTube playlist where viewers suggest alternatives.

Concepts covered:

  • Implementing Page Object Model.
  • Implementing Selenium Webdriver to perform actions on web element.
  • Implementing Mocha as the test framework and Mochawesome as the test reporter.
  • Implement Chai for better assertions (than NodeJS).
  • Implement parallel testing with Mocha.
  • Implement headless browser.

Selenium-Cucumber
REST-Assured Test Framework

This is a Proof-of-Concept test framework written in Java that uses Selenium Webdriver, Cucumber, and REST-Assured.
It's designed to test the Pokemon website but the API Testing will be done on the PokeAPI using the JSON Server.

This goes over the basic features of Selenium Webdriver, Cucumber, and REST-Assured.

  • Selenium Webdriver - a simple and concise programming interface that drives the browser via object-oriented API.
  • Cucumber - a BDD tool. It reads executable specifications written in plain text and validates that the software does what those specifications say.
  • REST-Assured - a tool for testing and validating REST services in Java.

This tutorial was done on Mac OS. I don't have an explicit tutorial for Windows. But, I'll post a YouTube playlist where viewers suggest alternatives.

Concepts covered:

  • Implementing Page Object Model.
  • Implementing Selenium Webdriver to perform actions on web element.
  • Implementing JUnit as the test framework.
  • Implementing Cucumber for writing specifications.
  • Implementing REST Assured for testing REST Calls.
